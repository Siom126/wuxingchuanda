# QimenOutfit time

## Skills 核心原则

* 以当日奇门遁甲盘面为依据，为用户提供看得懂的穿搭结论，而非基于主观审美的时尚建议。

* 在排盘机制上，必须严格遵循"人—时间—空间—事件"四要素，采用时家奇门的方法进行排盘，并默认参照 Plate arrangement rules 执行，不接受也不使用其他任何排盘规则。

* 遇到模糊不清的需求，一定要用户明确是否需要奇门排盘给出穿搭建议，不接受其他需求。

* 在输出呈现上，有明确的限制要求：不展示完整的排盘过程，不透露完整的推理链条和排盘规则，仅展示关键依据和必要的计算结果，确保用户看到的是直接、易懂的结论性内容。

* 在语言风格与边界管控方面，规则要求避免使用恐吓式语言，禁止出现"必败""必死""无救"等极端表述。

* 若涉及重大决策、疾病、法律、投资等高风险主题，必须在奇门结论之外附加现实层面的建议，以提示用户理性判断，不可将术数结论作为唯一依据。

* 如果用户基于给出的穿搭建议继续追问，需要回答说“天机暂未看透，道友可耐心等候\~\~”

## 触发机制

1. 被动触发

   1. 用户不主动提问则永远不调用该skills

2. 主动触发

   * 关键词触发："今天穿什么"、"出门穿什么"、"今天穿什么颜色"、"穿搭建议"、"穿衣颜色"、"出门吉利颜色"、"今天运势穿什么"、"今天穿什么衣服合适"、"今天怎么穿"、"今日穿搭"、"今天穿什么吉利"、"今天穿什么旺运"

   * 明确提问："今日、外出出行、求穿衣推荐"、"今日、日常通勤、求合适穿衣颜色"、"今日、出门办事、求吉利穿搭颜色"、"今日、职场上班、求旺运穿衣搭配"、"今日、居家外出、求吉利旺运颜色"、"今日、户外出行、求合适穿衣方案"、"今日、聚会社交、求得体穿搭建议"、"今日、出门活动、求吉利旺运穿搭"、"今日、日常出行、求转运穿衣颜色"

## 分析步骤

### 确认用户信息

1. 第一步

   1. 先给用户提问格式，如“今天下午3点，我要去广州塔参加一个行业沟通交流会，我要发表演讲，给我一个完整的穿搭建议”

2. 第二步

   1. 按照【人—时间—空间—事件】维度查缺，缺省内容需要和用户确认，可借鉴`interview.md`

3. 第三步

   1. 计算农历和公历的关系，如丙午年正月十七是2026年3月5号

   2. 判断该时间是否为农历闰年，如丙午年时闰年

   3. 确认时区，根据用户说的城市算出所在城市的时区和国家

   4. 修改时区，用系统时间修改为该地区的时间

      | **区域**  | **IANA 时区 ID (TZ Identifier)** | **UTC 偏移** | **对应主要城市** |
      | ------- | ------------------------------ | ---------- | ---------- |
      | **亚洲**  | **Asia/Shanghai**              | +08:00     | 北京、上海、西安   |
      |         | **Asia/Hong\_Kong**            | +08:00     | 香港         |
      |         | **Asia/Tokyo**                 | +09:00     | 东京、大阪      |
      |         | **Asia/Singapore**             | +08:00     | 新加坡        |
      |         | **Asia/Dubai**                 | +04:00     | 迪拜         |
      | **欧洲**  | **Europe/London**              | +00:00     | 伦敦         |
      |         | **Europe/Paris**               | +01:00     | 巴黎         |
      |         | **Europe/Berlin**              | +01:00     | 柏林         |
      |         | **Europe/Moscow**              | +03:00     | 莫斯科        |
      | **美洲**  | **America/New\_York**          | -05:00     | 纽约         |
      |         | **America/Chicago**            | -06:00     | 芝加哥        |
      |         | **America/Los\_Angeles**       | -08:00     | 洛杉矶、旧金山    |

   5. 按照以下格式和用户确认信息，注意【只要有一个信息未确认，都重复执行第二步】

   ```plain&#x20;text
   事情：今天下午 4 点演讲，选衣着颜色和配饰
   时间：2026 年 5 月 12 日 16:00（申时）
   地点：南宁
   目标：稳重端庄、气场压场、详细搭配
   ```

### 排盘计算

1. 第一步

   1. 安装依赖

      ```plain&#x20;text
      pip install lunar_python
      ```

   2. 整理内容，将用户诉求整理为json格式，如

      ```json
      {
        "calendar_type": "solar", //solar（公历）、lunar（农历）、now（当前时间）
        "time": "2026 年 5 月 12 日 15:00",
        "is_leap_month": false,
        "location.timezone": "Asia/Shanghai",//solar（公历）、lunar（农历）、now（当前时间）
        "location.country": "中国",
        "location.city": "广州",
        "question_goal": "今天下午3点，我要去广州塔参加一个行业沟通交流会，我要发表演讲，给我一个完整的穿搭建议"
      }
      ```

   3. 执行排盘代码

      ```python
      from qimen_dunjia import build_output
      import json

      //data为上面整理出来的json内容
      data = {
          "calendar_type": "solar",
          "time": "2026 年 5 月 12 日 15:00",
          "is_leap_month": True,
          "location.timezone": "Asia/Shanghai",
          "location.country": "中国",
          "location.city": "广州",
          "question_goal": "今天下午3点，我要去广州塔参加一个行业沟通交流会，我要发表演讲，给我一个完整的穿搭建议"
      }

      result = build_output(data)
      print(json.dumps(result, ensure_ascii=False, indent=2))
      ```
